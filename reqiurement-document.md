# Video Subtitle Generator 需求文档

## 1. 项目概述

### 1.1 产品定位
**Video Subtitle Generator** 是一款专业的视频字幕自动生成工具，基于最先进的 AI 语音识别技术，为视频文件自动生成与时间轴精确同步的字幕文件，支持 **SRT** 和 **ASS** 两种主流字幕格式。

### 1.2 核心价值
- **高效**: 一键批量处理，大幅减少人工字幕制作时间
- **精准**: 先进的 ASR 模型与后处理算法，保障字幕质量
- **易用**: 简洁直观的图形界面与命令行工具，适合不同用户群体
- **灵活**: 支持多种质量模式、语言选项与自定义配置

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 视频输入处理
- 支持主流视频格式（MP4、MKV、AVI、MOV、WMV 等）
- 支持单视频或批量多视频导入
- 支持拖拽添加视频文件（GUI）
- 视频文件路径有效性校验

#### 2.1.2 音频提取与预处理
- 自动从视频中提取音频流
- 音频标准化：16kHz 采样率、单声道、WAV 格式
- 音频增强配置：
  - `off`: 不进行音频增强
  - `voice`: 人声优化（轻度）
  - `strong`: 人声强化（重度，含高通/低通滤波、压缩、响度归一化）
- 自动降级：音频增强滤镜不可用时自动回退到无滤镜模式

#### 2.1.3 语音识别（ASR）
- 支持多 ASR 引擎适配器架构
- 当前实现：
  - `mock`: 模拟识别器（用于测试）
  - `faster_whisper`: 基于 faster-whisper 的高性能识别器
- 支持语言检测与指定：
  - 自动检测（`auto`）
  - 中文（`zh`）
  - 英语（`en`）
  - 日语（`ja`）
  - 西班牙语（`es`）
  - 法语（`fr`）
  - 俄语（`ru`）
  - 德语（`de`）
  - 意大利语（`it`）
  - 葡萄牙语（`pt`）
- 支持模型配置：
  - 模型选择（如 `large-v3`、`large-v3-turbo`）
  - 本地模型目录支持
  - 设备选择速度优先，仅支持GPU（CPU过于慢）
  - 计算精度配置

#### 2.1.4 质量模式预设
提供四档质量模式，一键应用优化配置：

| 模式 | 定位 | 特点 |
|------|------|------|
| `pro` | 专业精修 | 最高识别精度 + 最强后处理 + 异常片段自动细化 |
| `quality` | 最终成片 | 高质量识别 + 严格去重复 + 词级时间戳 |
| `balanced` | 默认模式 | 识别质量与速度平衡 |
| `speed` | 快速草稿 | 降低精度要求，提升处理速度 |

#### 2.1.5 VAD（语音活动检测）配置
- VAD 过滤开关
- VAD 预设配置：
  - `voice_focus`: 人声优先（高敏感度）
  - `balanced`: 平衡模式
  - `noise_robust`: 抗噪模式（低敏感度）
  - `fast`: 快速模式
- 可微调参数：
  - `voice_enhance_threshold`: 人声判定阈值（0.01-0.99）
  - `vad_min_silence_ms`: 最小静音时长
  - `vad_speech_pad_ms`: 语音片段前后填充
  - `vad_min_speech_ms`: 最小语音时长
  - `vad_max_speech_s`: 最大单段语音时长

#### 2.1.6 人声优先模板
一键应用针对复杂音频环境的优化配置：
- `quality_mode=pro`
- `audio_enhance_profile=strong`
- `vad_profile=voice_focus`
- 优化的 VAD 时长参数

#### 2.1.7 字幕后处理与质量保障


##### 时间戳质量约束
- 时间戳单调递增校验
- 单段时长有效性校验（`start_ms < end_ms`）
- 自动对齐与边界修复

#### 2.1.8 字幕输出
- **支持格式**：SRT 和 ASS 两种格式可选（默认为 SRT）
- **SRT 格式特点**：
  - 简单通用，兼容性最佳
  - 时间格式：`HH:MM:SS,mmm`（逗号 + 毫秒）
  - 文件编码：UTF-8
  - 文件名：与原视频同名，扩展名为 `.srt`
- **ASS 格式特点**：
  - 功能丰富，支持样式定义（字体、颜色、边框等）
  - 时间格式：`H:MM:SS.cc`（句点 + 厘秒）
  - 包含 Script Info、V4+ Styles、Events 三部分
  - 固定默认样式：Arial 字体、48 号、白色字幕、黑色边框
  - 文件编码：UTF-8
  - 文件名：与原视频同名，扩展名为 `.ass`
- **格式选择**：
  - GUI：通过下拉框选择输出格式
  - CLI：通过 `--subtitle-format` 参数指定
- **输出配置**：
  - 输出目录可配置
  - 覆盖策略可配置（是否覆盖已存在文件）
  - 批量处理时统一应用所选格式

### 2.2 高级功能

#### 2.2.1 模型缓存
- 相同配置的模型仅加载一次
- 支持多模型同时缓存
- 线程安全的缓存管理
- 提供缓存统计与清理接口



#### 2.2.3 配置持久化
- 用户配置自动保存到用户目录
- GUI 启动时自动加载上次配置
- CLI 支持 `--save-config` / `--load-config`
- 配置优先级：默认 &lt; 用户目录 &lt; 加载文件 &lt; 命令行覆盖

#### 2.2.4 模型下载与管理
- Hugging Face Hub 集成
- 本地缓存检测与状态可视化
- 离线模式支持
- 网络异常诊断与友好提示
- 私有/受限仓库访问支持（`use_auth_token`）

### 2.3 用户界面需求

#### 2.3.1 GUI 主界面
- **左侧面板**: 输入文件列表
  - 批量添加/移除文件
  - 文件列表显示
- **右侧面板**: 参数配置（分组卡片化）
  - 基础参数：Provider、Model、Language、Output
  - 高级参数：Quality Mode、Audio Enhance、VAD Profile、Voice Threshold、**Subtitle Format（字幕格式选择）**
  - 缓存状态可视化
- **任务操作区**:
  - 人声优先模板按钮
  - 开始/停止按钮
  - 进度条
- **结果展示区**:
  - 任务结果表格（含耗时统计）
  - 实时日志输出
- **执行过程可视化**:
  - 步骤日志（准备→提取音频→转写→修复→写入）
  - 阶段耗时统计

#### 2.3.2 CLI 命令行
- 完整的命令行参数支持
- 配置文件支持（`--config`）
- 配置保存/加载（`--save-config` / `--load-config`）
- **字幕格式选择**（`--subtitle-format srt|ass`，默认：srt）
- 结构化错误输出
- 进度与日志输出

## 3. 非功能需求

### 3.1 性能要求
- **模型加载**: 首次加载时间 &lt; 30s（large-v3-turbo 模型）
- **模型复用**: 缓存命中后，模型加载时间 &lt; 100ms
- **转写速度**: 
  - CUDA: &gt; 5x 实时（large-v3-turbo）


### 3.2 质量要求
- **SRT 格式合规性**: 100% 符合标准 SubRip 格式规范
- **ASS 格式合规性**: 100% 符合 ASS (Advanced Substation Alpha) 格式规范，包含完整的 Script Info、V4+ Styles、Events 部分

### 3.3 可靠性要求
- **优雅降级**: 依赖缺失时提供明确提示，不崩溃
- **异常处理**: 所有错误均映射为结构化错误码


### 3.4 兼容性要求
- **操作系统**: Windows 10+, macOS 10.15+, Linux（Ubuntu 18.04+）
- **Python 版本**: 3.9+
- **FFmpeg**: 支持系统 FFmpeg 或捆绑 FFmpeg
- **CUDA**: 可选，支持 CUDA 11.8+

### 3.5 可扩展性要求
- **ASR 引擎**: 适配器模式，易于接入新的识别引擎
- **配置**: 支持扩展新的配置项
- **后处理**: 模块化设计，便于添加新的后处理策略

## 4. 用户角色与使用场景

### 4.1 用户角色

| 角色 | 描述 | 典型使用方式 |
|------|------|-------------|
| 内容创作者 | 需要为视频快速生成字幕 | GUI + Quality 模式 |
| 视频编辑 | 需要精确字幕用于后期 | GUI + Pro 模式 |

| 测试/开发 | 需要调试或扩展功能 | CLI + Mock 模式 |

### 4.2 典型使用场景

**用户**: 内容创作者

1. 打开 GUI
2. 拖拽视频文件
3. 选择 `balanced` 模式
4. 点击"开始"
5. 等待完成，获取 Subtitle 文件

**用户**: 视频编辑
**步骤**:
1. 打开 GUI
2. 批量添加多个视频
3. 应用"人声优先模板"
4. 点击"开始"
5. 监控进度，获取所有 Subtitle 文件

#### 场景 3: 自动化流水线
**用户**: 技术用户
**步骤**:
1. 编写配置文件
2. 使用 CLI 批量处理
3. 集成到 CI/CD 流水线

## 5. 约束与假设

### 5.1 技术约束
- 依赖 FFmpeg 进行音频提取
- 依赖 faster-whisper 进行语音识别
- 模型文件较大（数百 MB 至数 GB），需要足够磁盘空间
- CUDA 加速需要 NVIDIA GPU 与正确配置的驱动

### 5.2 质量约束
- 识别准确率受音频质量影响
- 极低质量音频（高噪音、远场）可能需要人工修正
- 多语言混合场景可能需要指定语言

### 5.3 法律与合规
- 用户需确保拥有视频内容的合法使用权
- 模型使用需遵守相关许可（faster-whisper: MIT）
