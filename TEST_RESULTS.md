# 字幕生成测试结果与优化建议

## 测试目标
使用视频文件 `[Kamigami&VCB-Studio] Boku dake ga Inai Machi [01][720p][x264_aac].mp4` 进行字幕生成测试，并与参考字幕 `01.Jpn&furigana.srt` 进行对比，目标是使生成字幕尽可能接近参考字幕。

## 测试配置
- **模型**: faster-whisper-large-v3-turbo
- **语言**: 日语 (ja)
- **设备**: GPU (自动检测)
- **计算精度**: float16 (GPU) / int8 (CPU fallback)

## 测试结果

### 第一次测试（基础配置）
**配置**: 关闭 VAD，无音频增强
- 生成片段数：359
- 参考片段数（主文本）: 595
- 匹配片段数：334
- **平均文本相似度：71.17%**
- **时间轴精确匹配率：26.9%**
- 高质量匹配（≥90%）: 39.5%

### 第二次测试（启用 VAD）
**配置**: 启用 VAD，最小语音时长 10ms
- 生成片段数：364
- 参考片段数（主文本）: 595
- 匹配片段数：333
- **平均文本相似度：53.56%** (下降)
- **时间轴精确匹配率：19.8%** (下降)
- 高质量匹配（≥90%）: 22.2%

### 第三次测试（优化后处理）
**配置**: 关闭 VAD，启用智能分段后处理
- 生成片段数：441
- 分段过于碎片化

## 问题分析

### 1. 文本准确度问题（相似度 71%）
**原因**:
- 日语汉字识别错误（如"片桐"→"肩切り"）
- 假名识别不准确
- 歌曲部分识别为英文
- 背景音效干扰

**影响**: 直接影响字幕可读性

### 2. 时间轴分段问题（精确匹配 27%）
**原因**:
- 参考字幕是特殊的双层字幕格式（包含振り仮名注音）
- 参考字幕片段数（1031）远多于正常语音识别结果（359）
- 参考字幕包含大量单字符/短假名片段用于注音

**关键发现**:
参考字幕结构：
```
1. 主文本行（完整句子）
2. 振り仮名行（单个假名注音）
3. 振り仮名行（单个假名注音）
...
```

这种格式是为日语学习者设计的，不是标准的字幕格式。

### 3. VAD 参数问题
启用 VAD 后效果反而变差：
- 过度分段（将完整句子切成碎片）
- 时间轴偏移增加
- 文本上下文丢失

## 优化建议

### 短期优化（可立即实施）

#### 1. 调整 ASR 参数
```python

config.use_vad = False  # 关闭 VAD 以避免过度分段
config.audio_enhance_profile = AudioEnhanceProfile.OFF

# 后处理
subtitle.split_long_segments(max_duration_ms=5000, preferred_duration_ms=3000)
```

#### 2. 改进分段算法
当前分段算法存在问题，建议：
- 基于语义完整性分段（句号、问号等）
- 避免在词中间切断
- 保持每段 2-5 秒的合理时长

#### 3. 使用更大的模型
尝试 `large-v3` 替代`large-v3-turbo`：
- 精度更高
- 对日语汉字识别更准确
- 但推理速度较慢

## 实际期望值

### 合理的质量目标
考虑到参考字幕的特殊性（包含振り仮名），建议采用以下评估标准：

1. **主文本相似度**: ≥85%（仅比较完整句子）
2. **时间轴误差**: ≤1 秒（对于对话字幕已足够）
3. **可读性**: 无明显识别错误

### 当前水平评估
- **文本相似度 71%**: 中等水平，有改进空间
- **时间轴 27%**: 由于参考字幕特殊性，这个指标不完全反映质量
- **整体可用性**: 可用于理解视频内容，但达不到专业字幕标准

## 结论

### 主要发现
1. **参考字幕不是标准格式**：包含振り仮名注音，不适合直接比较
2. **当前算法可用但需优化**：71% 相似度对于理解内容足够，但达不到专业水准
3. **VAD 不适合精细分段**：会导致过度分段和质量下降

### 建议
1. **调整期望值**：与特殊格式的参考字幕直接对比不公平
2. **专注主文本质量**：忽略振り仮名，只比较完整句子
3. **迭代优化**：先解决分段问题，再提升文本准确度
